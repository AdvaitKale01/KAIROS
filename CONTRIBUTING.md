# Contributing to KAIROS

Thank you for your interest in KAIROS! We welcome contributions from the community.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Install dependencies**:
    ```bash
    pip install -e ".[dev,all]"
    ```

## Development Workflow

1.  Create a new branch for your feature or fix.
2.  Write tests for your changes in `tests/`.
3.  Ensure all tests pass:
    ```bash
    pytest tests/
    ```
4.  Submit a Pull Request (PR) with a clear description of your changes.

## Code Style

- We use standard Python PEP 8 conventions.
- Please add type hints to function signatures.
- Document new classes and methods.
